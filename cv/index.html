Profile

Documentation fails in predictable ways. It drifts from the product. It loses its owner. It answers the question nobody asked. I design systems that stop that from happening.

My background in creative writing means I arrived at technical writing already knowing how to earn a reader’s attention before asking for it. That is not a soft skill. It is an information architecture skill, and it is exactly what most technical documentation is missing.

I specialise in documentation systems for DevTools and SaaS teams: Docs-as-Code infrastructure, API reference documentation, developer-facing guides, and the structural work underneath that keeps documentation accurate as the product evolves.
